## 快速开始

### 安装要求

- **Flutter SDK** ^3.12.0
- **平台**：Android / iOS / Linux / macOS

### 系统依赖

**Linux (Fedora)：**

```bash
sudo dnf install clang ninja-build libsecret-devel gtk3-devel
```

**Android：**

需设置 `ANDROID_HOME` 环境变量：

```bash
export ANDROID_HOME="$HOME/Software/android-sdk"
export PATH="$PATH:$ANDROID_HOME/platform-tools"
```

### 构建运行

```bash
flutter pub get
flutter run                    # 连接到设备后运行
flutter run -d linux           # Linux 桌面
flutter run -d <device-id>     # 指定设备
```

首次运行会创建 SQLite 数据库和存储目录，无需额外配置即可浏览使用。

### 构建 APK

```bash
flutter build apk --debug      # 调试 APK（可直接安装）
flutter build apk --release    # 发布 APK（需签名）
flutter build appbundle        # AAB 格式（Google Play）
```

构建产物：`build/app/outputs/flutter-apk/app-debug.apk`

> 如果连接了 Android 真机（USB 调试开启），也可通过 `flutter install` 直接安装。

### 首次使用流程

1. 打开应用 → 进入空图库
2. 点击右上角 **+** 按钮导入图片
3. 开启 OCR 和 LLM 自动分析（可选，见设置）
4. 等待分析完成 → 使用颜色搜索查找图片

## 图片导入

支持常见图片格式：JPEG、PNG、WebP、GIF、BMP。

### 导入方式

点击图库右上角的导入按钮（`+`）进入导入页面：

```
操作步骤：
1. 点击「从相册选择」选取图片（可通过文件选择器多选）
2. 查看已选文件列表
3. 点击「导入」按钮开始导入
```

### 导入逻辑

- **去重**：对每张图片计算 SHA256 哈希，已存在的图片自动跳过
- **自动分析**：导入成功后自动将图片加入分析队列（需开启对应功能）
- **存储**：图片按 `年月/哈希值.扩展名` 结构存储，避免重名冲突

### 导入结果

导入完成后会显示结果摘要：

| 项目 | 说明 |
|---|---|
| 成功导入 | 新添加的图片数量 |
| 跳过 | 因重复而跳过的图片数量 |
| 错误 | 导入失败的图片路径及错误原因 |

## 颜色搜索

按颜色查找图片——通过 HSV 色彩选择器选取目标颜色，图库中返回该颜色相近的图片。

### 操作步骤

1. 点击图库顶部搜索按钮（🔍）进入搜索页面
2. 通过三个滑块调整目标颜色：
   - **色相**（0–360°）：决定颜色种类（红/橙/黄/绿/蓝/紫等）
   - **饱和度**（0–100%）：颜色的鲜艳程度
   - **明度**（0–100%）：颜色的明亮程度
3. 色块预览实时显示当前选中的颜色
4. 点击「搜索」按钮

### 搜索结果

- 按颜色相似度（ΔE 色差）从高到低排列
- 同一图片若有多个相近颜色，只显示一次
- 每页最多返回 50 张

### 算法说明

后台采用 CIE Lab 色彩空间的 ΔE（CIE76）色差计算，比 RGB 直接比较更符合人眼感知。

## 自动分析管线

导入图片后，系统自动按以下顺序处理：

```
导入 → 分析队列 → 颜色提取 → OCR 文字识别 → LLM 标签和描述
```

分析队列最多同时处理 2 张图片，轮询间隔 3 秒。

### 颜色提取

- 提取图片中的主色调（基于 K-Means 聚类量化）
- 转换为 CIE Lab 色彩空间值存储
- 用于颜色搜索

### OCR 文字识别

基于 Google ML Kit Text Recognition，支持中文和英文。

- 开启后导入的图片自动识别其中的文字
- 识别结果拆分为单词/词组作为 OCR 标签
- 识别结果也会传递给 LLM 做进一步分析

### LLM 标签与描述

需要配置 LLM 服务（见下方 LLM 配置）。

- **标签生成**：根据 OCR 结果生成 3∼5 个语义标签（逗号分隔）
- **描述撰写**：一句话概括图片内容（20 字以内）

标签和描述会保存在数据库并关联到对应图片。

## LLM 配置

支持两种 LLM 供应商：

| 供应商 | 默认地址 | 默认模型 | 说明 |
|---|---|---|---|
| Ollama（本地） | `http://localhost:11434` | `llama3.2` | 本地运行，免费，推荐 |
| OpenAI 兼容 | 自定义（如 `https://api.openai.com/v1`） | `gpt-4o-mini` | 需 API Key |

配置项：

| 参数 | 说明 |
|---|---|
| 供应商 | `ollama` 或 `openai` |
| 接口地址 | API 基础 URL |
| API Key | OpenAI 模式需要 |
| 模型 | 使用的模型名称（与供应商匹配） |

> 当前 LLM 配置通过代码或 provider 设置，后续版本将在设置界面中开放可视化配置。

## S3 云同步

支持将图片同步到 S3 兼容的对象存储（如 AWS S3、MinIO、Cloudflare R2 等）。

### 配置

在设置页面的「S3 云同步」卡片中配置：

| 参数 | 必填 | 说明 |
|---|---|---|
| Endpoint | ✅ | 存储服务地址，如 `s3.amazonaws.com` |
| Bucket | ✅ | 存储桶名称 |
| Region | ❌ | 区域，默认 `us-east-1` |
| Access Key | ✅ | 访问密钥 |
| Secret Key | ✅ | 密钥 |
| 使用 SSL | ❌ | 默认启用 |

配置完成后点击「同步到云」按钮开始同步。

### 同步功能

- **上传**：将本地图片上传到 S3 存储桶
- **下载**：从 S3 拉取图片到本地（开发中）
- **进度**：显示当前同步进度百分比

> 密钥信息通过 flutter_secure_storage 安全存储。

## 常见问题

**Q: 为什么搜索结果不准确？**

颜色搜索基于图片的主色调（前 5∼8 种颜色），不是像素级匹配。如果图片没有经过分析管线，颜色数据为空就无法搜索到。

**Q: 如何让图片被搜索到？**

确保在设置中开启 OCR 和 AI 标签，然后重新导入图片。已导入的图片需要等待分析队列自动处理。

**Q: 支持哪些语言？**

应用界面目前为英文，OCR 支持中文和英文。文档和代码注释以中文为主。
