# MemeHelper 性能预算

> 所属项目: MemeHelper
> 文档编号: 10-performance-budget.md
> 用途: 开发和测试阶段的性能基准

---

## 1. 启动性能

| 场景 | 目标 | 测量方式 | 备注 |
|------|------|---------|------|
| 冷启动到 Gallery 显示 | <2s | `flutter run --trace-startup` | 不含模型加载 |
| 热重载 | <1s | 默认 | - |
| 数据库初始化 | <300ms | Stopwatch | 首次创建或打开已有 DB |
| sqlite-vec 扩展加载 | <100ms | Stopwatch | 伴随 DB 初始化 |

**启动时不做的事**:
- 不自动加载 LLM 模型（用户触发分析时才加载）
- 不自动执行同步（在后台延迟执行）
- 不扫描文件系统

## 3. 分析管线性能

| 步骤 | 模型 | 目标耗时 | 说明 |
|------|------|---------|------|
| 颜色提取 | - | <200ms | 100x100 缩放 + 中值切割 |
| OCR (ML Kit) | - | <500ms | 中英混合，取决于图片文字量 |
| LLM 多模态描述 | Moondream 2B Q4 | <10s | 移动端推理 |
| LLM 多模态描述 | LLaVA 7B Q4 | <20s | 较大模型 |
| Embedding 生成 | MiniLM L6 Q4 | <300ms | 384 维向量 |
| 完整管线（含 LLM） | Moondream | <12s | 含 I/O 开销 |
| 完整管线（轻量模式） | 无 LLM | <1s | 仅颜色+OCR+embedding |
| 单 isolate 峰值内存 | Moondream | <200MB | 包含模型加载 |
| 单 isolate 峰值内存 | 仅 MiniLM | <50MB | CPU 使用率低 |

### 3.1 乐观/悲观分析

```
场景: 首次导入 100 张 meme

乐观（轻量模式）:
  100 × 1s = 100s (~1.5 分钟)  全部入搜索

悲观（含多模态 LLM，2 并发）:
  100 × 12s / 2 = 600s (~10 分钟) 全部分析完成
  用户导入后 10-15 秒第一张 meme 进入搜索

用户感知:
  导入完成 → 立即出现在 Gallery
  ~15s 后第一张变为绿色(分析完成)
  ~10 分钟后全部完成
```

| 多选模式切换 | 无卡顿 | 肉眼观察 |
| 页面转场动画 | 60fps | DevTools |

### 5.1 渲染优化策略

| 场景 | 方案 |
|------|------|
| 网格图片过多 | 使用 `ImageCache` 控制缓存上限（200MB），延迟加载不可见项 |
| 大图加载 | 缩略图缓存（256×256），详情页才加载原图 |
| 列表重建 | `ListView.builder` + `const` widget + `key` 保持状态 |
| 频繁刷新 | Riverpod `autoDispose` + debounce（搜索 300ms） |
| 动画 | 避免 `AnimatedBuilder` 嵌套过深，使用 `RepaintBoundary` |

## 6. 内存预算

| 组件 | 常驻内存 | 峰值内存 | 说明 |
|------|---------|---------|------|
| Flutter Engine | ~30MB | ~40MB | 框架自身 |
| SQLite | ~5MB | ~10MB | 运行时缓存 |
| 图片缓存 | ~50MB | ~200MB | 滚动中的缩略图 |
| Embedding 模型 (MiniLM) | ~50MB | ~50MB | 加载后常驻 |
| 多模态模型 (Moondream) | ~1.2GB | ~1.5GB | 仅在使用时加载 |
| 多模态模型 (LLaVA 7B) | ~4.5GB | ~5GB | 内存不足时不建议 |
| 分析 Isolate | ~10MB | ~200MB（带模型） | 分析完毕后释放 |
| **总计（轻量）** | **~135MB** | **~300MB** | 不使用多模态 LLM |
| **总计（Moondream）** | **~1.3GB** | **~1.8GB** | 正常使用 |
| **总计（LLaVA 7B）** | **~4.6GB** | **~5.5GB** | 需要 8GB+ 设备 |

## 7. 存储预算

| 数据类型 | 单条大小 | 总计 (50,000 meme) | 说明 |
|---------|---------|-------------------|------|
| Meme 图片（平均 200KB） | ~200KB | ~10GB | 主要存储占用 |
| SQLite 数据库 | - | ~100MB | 含向量数据 |
| 缩略图缓存（256×256） | ~15KB | ~750MB（全量） | 可按需清理 |
| Embedding 模型 (MiniLM) | ~45MB | ~45MB | 常驻 |
| 多模态模型 (Moondream) | ~1.2GB | ~1.2GB | 可选下载 |
| 多模态模型 (LLaVA 7B) | ~4.5GB | ~4.5GB | 可选下载 |
| **总计（轻量）** | - | **~11GB**（含 10GB meme） | |
| **总计（Moondream）** | - | **~12.2GB** | |

### 7.1 存储阈值警告

```
剩余空间 < 2GB      → 提示"存储空间不足"
剩余空间 < 1GB      → 停止导入，阻止模型下载
剩余空间 < 500MB    → 停止分析管线（写缓存可能失败）
```

## 8. 电量消耗

| 操作 | 估算耗电 | 说明 |
|------|---------|------|
| 分析一张（Moondream） | ~0.5% 电池 | 10s 推理，CPU 满载 |
| 分析一张（轻量模式） | ~0.05% 电池 | 1s 完成 |
| 批量分析 100 张 | ~50% 电池 | 不建议一次性大量分析 |
| 日常浏览 5 分钟 | ~1% 电池 | 常规 UI 渲染 |
| 同步 100 张（WiFi） | ~0.5% 电池 | 网络传输 |

**省电策略**:
- 仅在充电 + WiFi 时执行批量分析
- 非充电时限制分析速度为每分钟 1 张
- 低电量模式（< 20%）暂停后台分析

## 10. 性能测试执行

```bash
# 1. 启动跟踪
flutter run --trace-startup --profile

# 2. 分析启动时间
# 查看 build/run/startup_info.json 中的
# - timeToFirstFlutterFrame
# - timeToAppDraw

# 3. 性能图层检测
# 在 Profile/Release 模式下运行
flutter run --profile

# 然后通过 DevTools 的 Performance 页面检测帧率

# 4. 内存分析
flutter run --profile
# DevTools → Memory 页面

# 5. APK 大小
flutter build apk --release --target-platform android-arm64
ls -lh build/app/outputs/flutter-apk/app-release.apk
# 目标: <30MB（不含模型文件）
```
