# 搜索页面重设计

> 日期：2026-06-30
> 状态：已批准实施

## 设计目标

1. 颜色选择器从三段 HSV 滑块改为预设色板 + 自定义 HSV 调色盘
2. 支持多条件叠加搜索（颜色 + 文本同时生效）
3. 新的搜索页面 UI

## 核心交互

- 搜索栏 + 颜色选择器同时显示，各自独立
- 输入文本 → 语义/关键词匹配
- 点选色块 → 添加颜色条件
- 两个条件叠加过滤，结果自动更新
- 色板支持多色选择，色块以 chip 形式展示已选颜色

## 搜索级别自动降级

| 级别 | 能力 | 条件 |
|------|------|------|
| L3 | 语义 + 颜色 + 关键词 | 有 Embedding 模型 |
| L2 | 关键词 + 颜色 | 有 OCR/LLM 标签数据 |
| L1 | 仅颜色 | 无文本数据 |
| L0 | 浏览（全部） | 无任何条件 |

## 组件

- `SearchBar`：文本输入，300ms debounce
- `ColorPickerPalette`：预设色板（~24 色圆形色块），多选
- `ColorPickerDialog`：自定义 HSV 调色盘（水平色相条 + 2D 饱和度/明度区域）
- `ActiveFilters`：已选颜色 chip 列表，可逐个删除
- `SearchResultsGrid`：3 列网格，匹配度角标
- `SearchLevelIndicator`：当前搜索级别展示

## 搜索结果排序

颜色结果 × 文本结果加权合并：
- 仅有颜色 → 按 ΔE 色差升序
- 仅有文本 → 按匹配度降序
- 颜色 + 文本 → 加权合并排序

## 技术实现

- `SearchService` 新增 `search(String query, List<Color>? colors)` 方法
- 文本搜索：关键词→tags 表 LIKE 匹配，文件名匹配
- 颜色搜索：复用现有 `searchByColor`，支持多色
- 结果合并：加权评分排序
